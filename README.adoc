== The Paradigm Shift

For two decades, we've worried about "black box" AI and opaque systems. **We've been looking in the wrong direction.**

**The inversion**:
- Our boxes are **transparent** (we built the databases, models, systems)
- Reality is **dark** (unmeasured domains, blind spots, unknown unknowns)
- The question isn't "How does this model work?" but **"What haven't we measured?"**
- Not "Is the cat dead or alive?" but **"Are there cats beyond?"**

**metadata-grammar** is the cartographic standard for exploring this darkness:

- Catalog the **white box** (phenomenal databases we've built)
- Mark the **edges** (coverage boundaries, blind spots)
- Acknowledge the **darkness** (epistemic humility about the unknown)
- Guide **exploration** (where to step into darkness next)

**See link:PHILOSOPHY.adoc[PHILOSOPHY.adoc] for the complete framework.**

== What is Metadata Grammar?

Metadata Grammar goes beyond vocabulary to provide a **compositional grammar** for dataset metadata:

- **Vocabulary**: Terms and definitions (what VoID/Dublin Core provide)
- **Grammar**: Vocabulary + syntax + composition rules + validation + formal semantics + cartographic guidance

=== Key Features

✓ **Unified framework** - Integrates VoID (technical), Dublin Core (descriptive), DCAT (catalog)
✓ **Formal semantics** - Idris2 dependent types prove metadata properties
✓ **Modern concerns** - Data ethics, AI/ML metadata, sustainability, privacy
✓ **verisimdb integration** - Temporal versioning, immutable history, validation tracking
✓ **Multi-format** - RDF/Turtle, JSON-LD, JSON Schema
✓ **Legacy compatibility** - Automatic crosswalks to VoID/DC/DCAT

=== Example

[source,turtle]
----
@prefix mg: <https://hyperpolymath.org/ns/metadata-grammar#> .

:MyDataset a mg:Dataset ;
    mg:title "Example Dataset" ;
    mg:creator :JonathanJewell ;
    mg:tripleCount 1000000 ;
    mg:sparqlEndpoint <https://example.org/sparql> ;
    mg:validates-against :CoreShapes ;
    mg:validation-status mg:Valid .
----

== Integration with verisimdb

**verisimdb** provides temporal versioning for RDF graphs. Metadata Grammar integrates:

- Dataset metadata stored in verisimdb
- Provenance tracked via versioning
- Grammar rules enforced via SHACL
- Validation history preserved
